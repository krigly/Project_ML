# 🏦 Банковский маркетинг: прогнозирование депозитов

Анализ данных банковского маркетинга и построение моделей машинного обучения для прогнозирования открытия депозита клиентами.

---

## 📋 Описание проекта

Проект включает полный цикл работы с данными:
- **EDA** (разведочный анализ данных)
- **Предобработка** (очистка, кодирование, нормализация)
- **Построение моделей** (логистическая регрессия, решающие деревья, случайный лес, градиентный бустинг, стекинг)
- **Оптимизация гиперпараметров** (GridSearchCV, Optuna)
- **Оценка качества** моделей

---

## 📊 Данные

**Источник:** Маркетинговая кампания банка

**Целевая переменная:** `deposit` — согласился ли клиент открыть депозит

### Признаки:

| Категория | Признаки |
|-----------|----------|
| **Данные о клиенте** | age, job, marital, education, default, balance, housing, loan |
| **Данные о контакте** | contact, month, day, duration |
| **Прочие** | campaign, pdays, previous, poutcome |

## 🔄 Этапы работы

### Часть 1: Предобработка данных

- ✅ Обработка пропусков (медиана, мода)
- ✅ Преобразование `balance` (удаление символов, приведение к float)
- ✅ Удаление выбросов (IQR метод)
- ✅ Кодирование категориальных признаков
  - `education` → LabelEncoder
  - `job`, `marital`, `contact`, `month`, `poutcome` → One-Hot Encoding
- ✅ Создание возрастных групп

### Часть 2: Разведочный анализ (EDA)

- ✅ Визуализация распределений
- ✅ Корреляционная матрица
- ✅ Анализ выбросов
- ✅ Изучение категориальных признаков

### Часть 3: Преобразование данных

- ✅ LabelEncoder для `education`
- ✅ Бинарные переменные → 0/1 (`default`, `housing`, `loan`, `deposit`)
- ✅ One-Hot Encoding (`job`, `marital`, `contact`, `month`, `poutcome`)
- ✅ Отбор 15 признаков (SelectKBest)
- ✅ MinMaxScaler (нормализация)

### Часть 4: Построение моделей

| Модель | Параметры | F1 (Test) |
|--------|-----------|-----------|
| Logistic Regression | solver='sag', max_iter=1000 | ~0.81 |
| Decision Tree | max_depth=7 | ~0.82 |
| Random Forest | n_estimators=100, max_depth=10 | ~0.83 |
| Gradient Boosting | n_estimators=300, learning_rate=0.05 | ~0.83 |
| **Stacking** | DT + LR + GB | ~0.83 |

### Часть 5: Оптимизация гиперпараметров

- ✅ **GridSearchCV** — для логистической регрессии и дерева решений
- ✅ **Optuna** — для случайного леса
